## Hypotheses for next session (in priority order)

### 1. WDS BCD has code integrity hashes that no longer match (most likely)
When boot.wim was re-registered (Task #33), WDS regenerated the BCD it sends to clients. The BCD may now include integrity hashes or code integrity policy settings that reject the current EFI files. The EX file experiment may have compounded this — if WDS generated a BCD while the EX files were active, it might have hashed the EX versions.

**Test:** Run `wdsutil /Get-Server /Show:All` to check BCD/code integrity settings. Look for `BootProgram` and `PxePromptPolicy`.

### 2. Secure Boot DBX updated on IMAGE-ME (second most likely)
IMAGE-ME is a 2024 ThinkPad. If it received a UEFI firmware update via Windows (Microsoft pushed KB5025885-era DBX updates), the April 2024 bootmgfw.efi signing cert may now be revoked.

**Test:** **Temporarily disable Secure Boot on IMAGE-ME** in the UEFI firmware settings (F1 at boot → Security → Secure Boot). If PXE works with Secure Boot off, the issue is DBX revocation and the EX files are the fix.

### 3. WDS "no prompt" mode changed something
Task #34 disabled the "press enter" prompt. Depending on how this was implemented, it may have changed the WDS PXE response mode in a way that alters the BINL exchange.

**Test:** Check `wdsutil /Get-Server /Show:Config` for PxePromptPolicy setting.

### 4. BINL fix didn't actually take effect
Port 4011 was opened but maybe WDS BINL isn't actually listening on it, or the firewall rule profile doesn't match.

**Test:** `Get-NetFirewallRule -DisplayName "WDS BINL*"` and `Get-NetUDPEndpoint -LocalPort 4011`.

## WinRM is broken — fix first thing next session

WinRM accepts TCP connections on port 5985 but `Test-WSMan` hangs and `Invoke-Command` times out. Credentials are correct (Jason confirmed). The WinRM HTTP listener is apparently not responding to WSMan protocol despite the port being open.

**Fix — run this on pc-deploy console as Administrator:**
```powershell
Enable-PSRemoting -Force
Set-Item WSMan:\localhost\Service\Auth\Negotiate -Value $true
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" `
    -Name "LocalAccountTokenFilterPolicy" -Value 1 -Type DWord
Restart-Service WinRM
```

Then verify from ENG-2:
```powershell
Test-WSMan 192.168.5.141
```
